# Mobile UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make erid render properly on phones (portrait ≈360–430px) by scaling typography, spacing, the orb, and the features below it to the mobile ratio — without changing desktop or any logic.

**Architecture:** Pure responsive, mobile-first. The base (unprefixed) Tailwind values become the mobile sizes; the current desktop values are restored at `sm:` (640px) / `lg:` (1024px). Three repeated magic sizes are centralized into responsive tokens in `app/globals.css` (`.text-body`, a responsive `.font-editorial`, and a lower `--text-display` clamp floor) and applied at call sites. Same DOM, same components.

**Tech Stack:** Next.js 16, Tailwind CSS 4 (`@import "tailwindcss"` in `app/globals.css`), React 19. Visual verification via the running dev server + Playwright MCP.

## Global Constraints

- **Mobile-first:** base = mobile values; restore desktop at `sm:` (640px) / `lg:` (1024px). 640px is Tailwind's `sm` and the token media-query breakpoint — keep them aligned.
- **No logic / data / API changes.** Only `className`, inline `style`, and `app/globals.css`. The existing `npm test` suite (19 tests) must stay green.
- **Desktop (≥1024px) must render identically to today.** Every mobile change is a base value restored at `sm:`/`lg:`, or `hidden sm:block`.
- **No new dependencies, no structural/navigation rewrite** (no bottom tab bar / bottom sheet).
- **Tokens live in `app/globals.css`** alongside the existing `.font-hero` / `.font-editorial` / `.micro` classes.
- **Branch:** `feat/mobile-ui`.

## Task 1: Responsive type tokens in `app/globals.css`

**Files:**
- Modify: `app/globals.css` (`:root` `--text-display`, the `.font-editorial` rule, and a new `.text-body` rule)

**Interfaces:**
- Produces: CSS classes `.text-body` (17px → 24px at ≥640px, `line-height:1.5`, `letter-spacing:-0.24px`) and a responsive `.font-editorial` (30px → 48px at ≥640px); a lowered hero clamp `--text-display: clamp(34px, 8vw, 54px)`. Later tasks apply `.text-body` in place of `text-[24px] leading-normal tracking-[-0.24px]`, and drop inline `text-[48px]` from elements that already carry `font-editorial`.

- [ ] **Step 1: Lower the hero display clamp floor**

In `app/globals.css`, in the `:root` block, change:
```css
  --text-display: clamp(42px, 8vw, 54px);
```
to:
```css
  --text-display: clamp(34px, 8vw, 54px);
```
(Desktop still hits the 54px cap; only sub-~425px screens drop below 42px. `.font-hero` already consumes `--text-display`, so the hero headline becomes gentler on narrow screens with no other change.)

- [ ] **Step 2: Give `.font-editorial` a responsive size**

Replace the existing rule:
```css
.font-editorial {
  font-family: var(--font-instrument-sans), ui-sans-serif, system-ui, sans-serif;
  font-weight: 400;
  line-height: var(--leading-heading);
  letter-spacing: var(--tracking-heading);
}
```
with:
```css
.font-editorial {
  font-family: var(--font-instrument-sans), ui-sans-serif, system-ui, sans-serif;
  font-weight: 400;
  font-size: 30px;
  line-height: var(--leading-heading);
  letter-spacing: var(--tracking-heading);
}
@media (min-width: 640px) {
  .font-editorial {
    font-size: 48px;
  }
}
```
(Call sites that currently write `font-editorial text-[48px]` must drop the inline `text-[48px]` so this size wins — done in Tasks 4 and 6. A Tailwind `text-[48px]` utility would otherwise override this rule.)

- [ ] **Step 3: Add the `.text-body` token**

Add immediately after the `.font-editorial` block:
```css
.text-body {
  font-size: 17px;
  line-height: 1.5;
  letter-spacing: -0.24px;
}
@media (min-width: 640px) {
  .text-body {
    font-size: 24px;
  }
}
```

- [ ] **Step 4: Build + lint to confirm CSS is valid**

Run: `npx tsc --noEmit && npm run lint`
Expected: both exit 0 (no JSX/TS regressions; CSS isn't type-checked but a malformed file would break the next build).

- [ ] **Step 5: Visual sanity — hero headline eases on mobile**

Start/confirm dev server, then via Playwright at **390×844**: navigate, close the sources panel, screenshot the hero.
Expected: the headline "What should your notes become?" is visibly smaller than before (~34px vs ~42px) and no longer crowds the orb. Nothing else should change yet (no call sites use `.text-body` until later tasks).

- [ ] **Step 6: Commit**
